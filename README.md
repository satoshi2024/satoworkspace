**【2. DB登録処理（バックエンド）】**

 * **修正前：** 返戻情報をDBへ登録する際、帳票名が一律として固定（ハードコーディング）されていた。
 * **修正後：** 画面から渡される帳票IDを判定し、およびの正しい帳票名を動的にセットしてDBへ登録するロジックに変更した。
   ■
 * **修正前：** 既存ベーステーブル検索時のパラメータ不備（年度の誤用、和暦/西暦の不一致）により抽出結果が常に0件となり、登録処理がスキップされていた。また、ベーステーブルを持たない帳票の考慮が欠落していた。
 * **修正後：** 既存の共通ロジックに影響を与えないよう隔離メソッドを新設し、帳票ごとの分岐処理を追加した。
   * **410等（台帳データあり）：** 検索条件を是正し、取得した台帳データを利用して動的な帳票IDとともに正しくDBへ登録するよう修正。
   * **411（台帳データなし）：** 検索処理を迂回し、画面から取得可能な必須パラメータのみを直接構築してDBへ登録する専用ロジックを実装した。
